# Cafe Daneh — Frontend (React + Vite)

این، نسخه‌ی واقعیِ فرانت‌اندِ کافه دانه است — نه artifact کلاود، بلکه یک پروژه‌React استاندارد که با
**Vite** ساخته شده و به‌بک‌اند واقعی (پوشه‌`cafe-daneh-backend`) با`fetch` وصل می‌شود.

✅ **این پروژه واقعاً ساخته، اجرا، و در برابر یک بک‌اند واقعی تست شد** — ثبت‌نام، سفارش با تخفیف
محاسبه‌شده‌توسط سرور، باشگاه مشتریان، دعوت دوستان، و پنل مدیر (بدون PIN، با نقش واقعی کاربر) — همگی
به صورت end-to-end تأیید شدند.

## اجرا

نیازمندی: Node.js (هر نسخه‌ی نسبتاً جدید) و اینترنت (برای`npm install` یک‌بار).

```bash
# ۱) ابتدا بک‌اند را در یک ترمینال جدا اجرا کنید (به README آن پروژه مراجعه کنید)
cd ../cafe-daneh-backend && npm run seed && npm run dev

# ۲) در یک ترمینال دیگر، فرانت‌اند را اجرا کنید
cd cafe-daneh-frontend
npm install
npm run dev
```

سپس آدرس`http://localhost:5173` را در مرورگر باز کنید.

اگر بک‌اند روی آدرس یا پورت دیگری اجرا می‌شود، فایل`.env.example` را به`.env` کپی کنید و
`VITE_API_URL` را تغییر دهید.

## چه چیزی از artifact به این پروژه تغییر کرد؟

| | artifact (قبلی) | این پروژه (الان) |
|---|---|---|
| ذخیره‌سازی | `window.storage` (مخصوص کلاود) | API واقعی + پایگاه‌داده‌ی SQLite |
| محاسبه‌ی قیمت/تخفیف | در خود مرورگر | روی سرور — کلاینت فقط یک تخمین نمایش می‌دهد |
| ورود | یک فرم ساده، بدون رمز | ثبت‌نام/ورود واقعی با رمز هش‌شده و توکن امضاشده |
| پنل مدیر | پشت یک PIN ثابت در کد | پشت نقش واقعی کاربر (`role: 'admin'`)، بررسی‌شده در سرور |
| دعوت دوستان | شبیه‌سازی‌شده، فقط در یک دستگاه | واقعی، بین حساب‌های مختلف |
| عکس محصولات | base64 توی همان فایل | همان روش (ساده‌ترین برای این مرحله) — `src/photos.js` |

موارد زیر همچنان فقط محلی (در`localStorage` مرورگر) هستند، چون نیازی به سرور ندارند: تم ظاهری،
علاقه‌مندی‌ها، مرتب‌سازی منو، و خود سبد خرید (تا قبل از ثبت نهایی سفارش).

## ساختار

```
src/
  api.js              تنها فایلی که آدرس بک‌اند را می‌شناسد — بقیه‌ی کد از این صدا می‌زند
  pricingPreview.js    محاسبه ی تخمینی (فقط نمایشی) — مقدار واقعی همیشه از پاسخ سرور می‌آید
  photos.js            عکس های محصولات (base64)
  components/          DanehLogo, Bean
  App.jsx              کل رابط کاربری
  styles.css           طراحی بصری (همان سیستم رنگی/تایپوگرافی artifact قبلی)
```

## ساخت نسخه‌ی نهایی (Production Build)

```bash
npm run build
```

خروجی در پوشه‌ی`dist/` ساخته می‌شود — قابل آپلود روی Vercel، Netlify، یا هر هاست استاتیک دیگر.
فقط فراموش نکنید قبل از build، متغیر`VITE_API_URL` را به آدرس واقعیِ بک‌اند دیپلوی‌شده تغییر دهید.
